## 一、 项目基本信息
* **项目名称**：基于YOLOv8的移动城市巡检小车系统设计与实现
* **核心硬件**：STM32（底盘主控与环境感知）、ESP32-CAM（视频流传输与软网关预留）
* **核心软件**：Python (Tkinter, OpenCV, SQLite), YOLOv8n (视觉算法)
* **业务痛点**：传统城市巡检依赖人工、效率低、异常发现滞后、缺乏数据闭环。

## 二、 系统核心架构与创新点
本项目采用“感知 - 边缘 - 上位机”一体化架构：
1. **底层感知与执行 (STM32)**：负责超声波避障、红外循迹、DHT11温湿度采集、舵机云台控制及电机驱动。通过十六进制单字节指令（如 `b'@'` 巡逻, `b'H'` 避障）与上位机通信。
2. **边缘图传与网关 (ESP32-CAM)**：开启 81 端口吐出原生视频流，进行静态 IP 适配，并作为未来 MQTT 协议重构的“软网关”预留节点。
3. **上位机智能调度 (Python)**：
   * **UI 交互**：Tkinter 开发深色科技风面板，支持 WASD/QE 按键状态机控制、多模式一键切换（循迹/避障/自动巡逻）。
   * **AI 视觉引擎**：集成 YOLOv8 模型，**创新点**包括：实施针对性**分级置信度**（井盖0.85，坑洼0.20防漏检）；开发**“10秒防抖冷却机制”**防刷屏；新增支持本地照片上传的**通用模式离线识别模块**。
4. **业务闭环与持久化 (SQLite)**：摒弃单表流水账，构建具备 `session_id` 的一对多关系型数据库。AI触发警报时，实时融合“温湿度环境快照”；历史数据中心支持右键菜单更新工单状态（待处理/已修复/误报），实现完整的物联网派单闭环。

## 三、 数据库结构 (patrol_record_v2.db)
* **patrol_sessions (任务主表)**：`session_id`, `start_time`, `end_time`, `status`
* **sensor_logs (环境流水)**：外键关联 `session_id`，记录定时或突变的温湿度与距离。
* **detection_logs (事件图库)**：外键关联 `session_id`，记录识别目标、置信度、截图路径，融合 `snapshot_temp`/`humi`，并维护工单 `status`。

## 四、 论文后续演进规划（学术与工程深度补充）
为了提升毕业论文的学术含金量与商业落地价值，系统在概念上规划了以下升级（重点用于论文的“优化与展望”章节）：
1. **算法创新**：针对复杂光线下的坑洼误报，计划在 YOLOv8 骨干网络引入 CBAM 或 SE 注意力机制，并混合自采实景数据集进行微调。
2. **空间定位**：计划接入 ATGM336H 模块解析 NMEA 协议，为巡检事件补充 GPS 经纬度，解决“有病害无位置”的痛点。
3. **架构重构**：引入 MQTT 协议实现车路云解耦，将 ESP32 改造为软网关，同时保留原有蓝牙作为近场容灾备份链路。
4. **移动预警**：接入 Server酱/PushPlus 的 HTTP API，实现高危病害的微信/短信移动端实时报警推送。
