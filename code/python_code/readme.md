# 毕业设计中期检查进展汇报

**项目名称：** 基于 YOLOv8 的移动城市巡检小车系统设计与实现

## 1. 项目概述
本项目旨在开发一套“感知-边缘-上位机”一体化的城市智能巡检系统。通过 STM32 驱动传感器和底盘，通过 ESP32-CAM 提供视觉流，并在 Python 上位机端利用 YOLOv8 算法进行路面异常（如坑洼、井盖异常、易拉罐）的实时检测与自动留证，最终形成一套低成本、高效率的物联网巡检方案。

## 2. 目前已完成的工作与阶段性成果
截至中期检查，本项目已顺利打通底层硬件到上位机 AI 算法的完整链路，具体成果如下：

### 2.1 硬件与固件开发
* **STM32 主控逻辑**：已完成超声波避障、红外黑线循迹、DHT11 温湿度采集、舵机云台控制及电机驱动的底层 C 语言代码编写，小车可实现基础自主运动。
* **ESP32-CAM 视频图传**：完成视频流 Web Server 搭建，修改并适配了静态 IP 分配逻辑（应对校园网与手机热点环境），调整分辨率为 VGA(640x480) 以平衡画质与延迟，并成功打通了上位机的无极调光接口。
* **双向通信协议**：确立了上行（环境状态、距离字符串）与下行（Hex 十六进制单字节控制指令，如 b'@' 前进、b'B' 停止、b'Q' 云台升）的稳定串口通信协议。

### 2.2 上位机与 AI 算法集成
* **UI 交互控制台**：使用 Tkinter 开发了深色系专业控制台。实现了按键状态机（按下即走、松开即停），整合了视频监控、数据图表和运动控制模块。
* **YOLOv8 模型部署与调优**：
  * 成功将训练好的 best.pt 模型集成至视频流处理线程。
  * **[创新点] 差异化置信度算法**：针对不同目标的检测难点，代码中实施了分级阈值策略（易拉罐设为0.6，井盖要求极高确信度设为0.85，坑洼为防漏检设为0.20），大幅降低了误报率。
  * **[创新点] 智能防刷屏机制**：开发了冷却字典（Cooldown Dict），针对同一类别的目标设定 10 秒冷却时间，避免视频帧连续触发导致数据库与硬盘爆满。

### 2.3 数据持久化模块
* 搭建 SQLite 数据库，实现环境数据（距离、温湿度）基于“突变阈值”和“心跳包”的智能筛选存库。
* 开发了 AI 自动留证功能，将截图文件归档至 `data_evidence` 目录，并记录置信度与时间戳。

## 3. 当前存在的主要问题与难点
1. **数据孤岛问题**：当前数据库记录缺乏宏观的“巡检批次”概念，环境数据与 AI 警报数据未建立强关联，数据挖掘价值受限。
2. **网络环境适应性**：ESP32-CAM 在校园网（AP隔离/复杂认证）下连接不稳定，目前通过手机热点固定静态 IP 解决，但对后续的通用化部署有一定限制。
3. **模型泛化能力**：由于坑洼（Pothole）阈值设置较低（0.2），在开启 OpenCV 图像锐化算法后，部分粗糙的地板纹理偶尔会被误报。
